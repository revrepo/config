RevAPM Product Terms and Definitions

 

Term

Description
95/5 percentile 

a traffic billing method used by Internet transit providers–see http://en.wikipedia.org/wiki/Burstable_billing for details

browser proxy (BP)

RevAPM proxy located close to end users (last mile proxy) and performing HTTP proxy and object caching functionality

cache, cache proxy

a RevAPM server (like BP) providing object caching functionality

cache hit

a cache hit occurs when the requested data can be found in a cache, while a cache miss occurs when it cannot

cache Time-To-Live (TTL)

how long a cached object is considered to be “fresh” and can be used/served without a revalidation

content optimizer (CO)

RevAPM proxy located close to customer origin servers (first mile proxy) and performing HTTP proxy and content optimization functions

core site

a RevAPM site hosting RevAPM management services like customer portal, databases, monitoring, management tools, etc

customer API, API

RevAPM customer API application designed to provide programmable access to RevAPM configuration management and traffic reporting functionality

customer origin, customer origin web server, origin web server (OWS)

a web server managed by a RevAPM customer and used to host/generate content served by the RevAPM platform

customer portal (CP)

RevAPM customer portal application designed to provide configuration management and traffic reporting functionality to RevAPM customers

disaster recovery (DR)

disaster recovery (site)–secondary RevAPM core site

Dynamically Optimized Transport System (DOTS)

RevAPM proprietary TCP congestion protocol compatible with standard TCP implementations

edge site, edge server

a RevAPM site/server located close to end users and hosting RevAPM BP/CO proxy servers

first mile (FM)

the Internet network part from customer origin websites to the closest RevAPM edge servers

GB Transfer (GBT)

a traffic billing method primary used for cloud services

HyperDrive (HD)

RevAPM proprietary technology for data communication between RevAPM first and last mile servers

last mile (LM)

the Internet network part from end users (cable/ADSL/mobile/Wifi) to the closest RevAPM edge servers

middle mile (MM)

the Internet network part between RevAPM edge servers serving an end user request

Mobile Acceleration Platform (MAP)

RevAPM’s proprietary solution for acceleration of content delivery for mobile apps and web properties

object invalidation

forced TTL expiration for a cache object, so on the request for the object a proxy will need to revalidate it with the customer origin server

object purge, object flush

removal of a customer-specified object from RevAPM cache servers

object Time-To-Live (TTL)

can be applied to DNS, edge and browser object caching

object, web object

 a web file pointed by an unique URL

page view

download of a full web page including all objects that construct the page

proxy, proxy server

a RevAPM server serving HTTP and RMP requests for end users

real user monitoring (RUM)

passively monitors and collects data from users to give insight into a website performance

real user monitoring (RUM) analytics

analysis and reports from the RUM

request, hit

request or hit is the same thing

request per second (RPS)

the number of requests or hits per second

RevAPM Mobile Protocol (RMP)

RevAPM proprietary data transfer protocol optimized for wireless last-mile and used between SDK-enabled mobile apps and RevAPM edge proxy servers

third-party object

any web object on a web page not served from RevAPM or customer web origin

Time To First Byte (TTFB)

see http://en.wikipedia.org/wiki/Time_To_First_Byte for details 

Varnish Configuration Language (VCL)

see https://www.varnish-cache.org/trac/wiki/VCL for details

Virtual CDN (VCDN)

a RevAPM functionality which allows to provision, manage and serve RevAPM customers on a subset of RevAPM production servers
