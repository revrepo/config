Execution Order of Caching Rules

It is important to understand the order in which rules are executed. The caching rules are interpreted in the order they are provided. Newer rules that match an object that are also matched by a previous rule will override the previous rule for that object. This means one must ensure that the rules are ordered from “more general” to “more specific”.

For example:

  • The first rule will match all JPEG images on the site.
  • The second rule will match all JPEG images in the /album directory.
  • The third rule will match all JPEG images in the /album/special directory.
