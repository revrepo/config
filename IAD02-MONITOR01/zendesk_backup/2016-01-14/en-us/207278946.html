SDK Quick Start Guide

The goal of this document is for the user to get the Rev SDK functional quickly.  The figure below is a high level diagram of the work flow.  

 

Step 1 - Sign in to the Rev portal (https://portal.revapm.net/) using your existing RevAPM portal credentials.



Step 2 - Select the Apps tab from the top of the portal page.  Choose the operating system from the left of the column (currently only iOS SDK is provided/supported).



Step 3 - Select Add new app button


Step 4 - Enter the App name, in this example below, the App is called “MyExampleApp”.


Step 5 - Once the App is registered on the Rev Platform, use the “Back to List” button.

Step 6 - The newly added App should appear in the iOS Apps List.   

Step 7 - Access the SDK key by clicking on the eye icon.  This will reveal the SDK key that will be used in the mobile App to initialize the Rev SDK code.

563c50ed-0e66-4a67-871d-1114b848bbb

Step 8 - Load the SDK package into Xcode

Step 9 -  For Objective C to initialize the SDK, the following two lines will need to be added to the mobile application code. 

   #import <RevSDK/RevSDK.h>

   [RevSDK startWithSDKKey:@"563c50ed-0e66-4a67-871d-1114b848bbb"];

Step 10 - Compile and deploy the mobile application.

Step 11 (Optional) - Perform a fine-tuning of the SDK configuration for the app. Please see “Rev SDK Configuration Guide” for details.

Step 12 - Use the “Mobile Analytics” section in the Rev customer portal to analyze the app data transfers and user base parameters.
