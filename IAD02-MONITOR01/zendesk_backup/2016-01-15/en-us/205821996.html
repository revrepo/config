RevAPM Caching Rules

A caching rule is a set of parameters that changes the caching behavior of one or more URLs that match a pattern.  The caching rule is specified either as a wildcard or a Perl-compatible regular expression. 

This section provides basic information to users who need to configure caching rules for RevAPM edge proxies. To configure caching rules, you need to be logged into the RevAPM Customer Portal as an Admin or as a user with configure privileges.

To modify the configuration rules:

  1. Log into the RevAPM Portal. 
  2. On the Web menu, click Domain in the side menu.
  3. Next to the domain that you want to edit the caching rules, click the Pencil icon and then the Advanced mode button.

You will see in the GUI either the default caching configuration in JSON for a domain with no prior caching configuration or the present caching configuration.

  1.  When you complete your editing, click Update.

After reviewing the following articles about object caching rules in RevAPM, access the RevAPM Customer Portal to modify the default (or previous) configurations.
